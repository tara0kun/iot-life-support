# 認知症祖母 IoTサポートプロジェクト 引き継ぎプロンプト

以下のプロジェクトの開発を手伝ってください。

---

## プロジェクト概要

認知症が進行している祖母の日常生活をIoT技術でサポートするシステムを構築します。

### 祖母について
- 認知症がかなり進行している（短期記憶が著しく低下）
- プライドが高く、指摘・命令に強く反発する
- 「自分はできている」という認識が強い
- 認知症以前は家事全般をすべてこなしていた
- 祖父・母と同居（ほぼ誰かがいるが、トイレや短時間の外出時に問題が起きる）

### 設計の絶対原則（最重要）
| ❌ やってはいけない | ✅ 正しいアプローチ |
|---|---|
| 「もう食べたよ」と直接指摘する | 本人が自分で確認できる記録を見せる |
| 「また炊いたの？」と責める | 機械の不調に見せて物理的に阻止する |
| 監視システムと分からせる | 「自分の記録帳」として提示する |
| 家族が止めていると思わせる | 「機械が壊れている」と自然に思わせる |

---

## 解決する3つの問題

### 🍚 問題1：ご飯問題（最優先）
- 数分目を離した隙に何度も炊飯してしまう
- 炊飯中に蓋を開けて食べようとする
- 生米の上にご飯を入れて再炊飯してしまう
- 鍋いっぱいに料理を作って食材ロスが発生

**解決方針（2段階）**

Phase 1（まず試す）：リビングのタブレットに「今日の記録帳」を表示し、「あ、もう炊いたんだ」と自分で気づいてもらう

Phase 2（Phase 1が難しければ）：Tapo P115で1回使用後に自動OFFロック（「機械の調子が悪い」と思わせる）＋2回目検知でLINE通知

---

### 🚪 問題2：訪問者問題
- 訪問販売・新聞勧誘などが来て不要な契約を結んでしまう
- 現在インターホンなし
- 来訪後すぐに忘れてしまう

**解決方針：**
- Tapo D230S1（スマートドアホン）で来客を自動撮影・記録
- 玄関そばのタブレットに「この人は3回来ています／前回〇〇を勧めてきました」と即時表示
- 祖母がボタンで用件を選択（新聞／宅配／知人／その他）
- 家族がスマホから内容を確認・修正できる
- 顔認識で過去の来訪者を自動照合

---

### 🚿 問題3：入浴・洗髪問題
- お風呂には入るが頭を洗わないことがほとんど
- 「洗った」と言い張る

**解決方針：**
- スマートプラグをドライヤーのコンセントに設置
- ボタンを押さないとドライヤーの電源が入らない
- 「誰が使いますか？」を大きなボタンで選択
- 対象者を選択→ドライヤー使用可能→「頭を洗いました」と記録帳に自動記録

---

## 開発環境

- **ハードウェア：** Raspberry Pi（SSH接続）
- **エディタ：** VS Code（SSH Remote）
- **スマートプラグ：** TP-Link Tapo P110M（電力監視＋遠隔制御、Matter対応）
- **センサー：** TP-Link Tapo T110（開閉）、T100（モーション）、H100（ハブ）
- **カメラ：** TP-Link Tapo C220（RTSP対応）

## 技術スタック

| レイヤー | 技術 |
|---|---|
| スマートプラグ制御 | python-matter-server (Matter) / python-kasa (KLAP) |
| バックエンド | Python（FastAPI）|
| フロントエンド | HTML/CSS/JS（Jinja2テンプレート）|
| 顔認識 | Python face_recognition |
| 通知 | LINE Messaging API |
| データ保存 | SQLite |
| 外部公開 | Cloudflare Tunnel |
| 中央管理 | Raspberry Pi |
